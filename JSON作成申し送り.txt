【申し送り｜臨床推論シミュレーションアプリ 症例作成フェーズ固定版・更新版】

救急救命士向け臨床推論シミュレーションアプリ開発を継続中。
現在は app_simulator_full.py の土台とUI整理が完了しており、ここからは 100症例作成を目標に、症例追加を最優先で進めるフェーズに入っている。

今後はこの申し送りを100症例到達まで継続使用する前提とし、ルールは毎回ぶらさず厳格に守ること。

────────────────
1. 現在の開発方針
────────────────
- アプリ本体の大改修フェーズはいったん終了
- 今後は症例拡充・内容強化フェーズ
- 主戦場は新規症例追加、既存症例修正、画像とJSONの整合
- app.py は触らない
- app_simulator.py は残す
- app_simulator_full.py を基準アプリとして使う

────────────────
2. 今後の作業順序（固定）
────────────────
今後は以下の順番を基本ルールとして固定する。

【作成順】
1. 先に現場イラストを作成
2. そのイラストに合わせてJSONを作成
3. 必要に応じて画像ファイル名・media指定を調整
4. 保存
5. ローカル確認
6. GitHub push
7. Streamlit反映

【理由】
- 先にイラストを作ることで、scene1の現場状況が具体化しやすい
- JSON側のmedia記述と整合しやすい
- 国試画像の直接流用感を避けやすい
- 症例の世界観を先に固定できる

────────────────
3. カテゴリ名（固定）
────────────────
症例JSONのカテゴリは以下で固定。
今後、申し送り・JSON作成・保存先指定では必ずこのカテゴリ名をそのまま使うこと。

- cardiovascular
- endocrine_metabolic
- environmental_special
- gastrointestinal
- neuro
- orthopedic
- psychiatric
- reproductive_obstetric
- respiratory
- special_population
- toxicology
- trauma
- urinary

勝手に和名へ変更しないこと。
別の英語名へ言い換えないこと。
例：circulatory などは使わない。

────────────────
4. 画像ファイル保存ルール（固定）
────────────────
画像・イラストファイルはすべて以下に保存する。

media/

例：
media/acute_heart_failure_home_scene1.png

cases/media ではなく、今後の基準は user 指定の media を優先すること。

【重要補足】
作成する画像は、問題部を読んだScene1の現場状況をイラストにする。
JSONでも、そのイラストを表示するコードにする。

────────────────
5. 画像作成ルール（今回の修正点・最重要）
────────────────
ここを今後絶対に間違えないこと。

【画像に入れてよいもの】
- 傷病者
- 室内、施設、自宅、路上などの背景
- 姿勢、表情、手足の動かしにくさ
- 食事中、倒れている、座り込んでいるなどの現場状況
- 病態を示唆する範囲の周辺情報

【画像に入れてはいけないもの】
- 問題文
- 選択肢
- 正解表示
- 解説文
- 文字情報全般
- 救急隊
- 問題集風のレイアウト
- 国試問題そのものの図解化

【固定ルール】
- 画像は「症例説明画像」ではなく、「現場の1シーン」を描く
- 主役は傷病者
- 救急隊は入れない
- 必要なら家族・介護士などの周辺人物は入れてもよいが、主役にしない
- 問題文の文字起こし画像にしない
- Scene1でそのまま表示される前提の自然なイラストにする

────────────────
6. JSON保存ルール（固定）
────────────────
- 1症例1JSON
- 保存先：
  cases/カテゴリ名/ファイル名.json

例：
cases/cardiovascular/case_cardiovascular_acute_heart_failure_pulmonary_edema_001.json

────────────────
7. JSON形式ルール（最重要）
────────────────
今後のJSONは app_simulator_full.py 用の新形式に厳格準拠する。

【必須ルール】
- sceneは7場面固定
- questions[] は使わない
- scene直下に必要項目を書く
- debriefing を必ず入れる

【各scene直下に置く項目】
- id
- title
- phase
- scene_goal
- ideal_flow
- ideal_score_delta
- action_tag
- text
- visible_data
- media
- type
- prompt

【選択形式】
- 単一選択・複数選択 → options
- 並べ替え → ranking
- テンプレ選択 → templates

【debriefing】
- summary
- ideal_actions
- good_points
- cautions

────────────────
8. 旧形式との違い（混同禁止）
────────────────
以前のズレの原因はここだったので、今後は厳守。

旧：
scene -> questions[0] -> type/options

新：
scene直下 -> type / prompt / options / ranking / templates

つまり、
questions配下は使わない。
これを崩すと app_simulator_full.py と不一致になる。

────────────────
9. media の書き方
────────────────
画像が必要なsceneでは、media に入れる。
不要なら [] にする。

画像を使う場合は、以下の考え方を守る。

- 国家試験の図版そのままは使わない
- 要点だけ残した再構成画像にする
- 構図・背景・人物配置を少し変える
- 重要所見が分かればOK

JSONでは以下を基本で入れる。

- file
- description
- source_note

source_note の基本文：
既存図版を直接使用せず、要点のみ残したオリジナル再構成画像

────────────────
10. 国試っぽさを薄めるルール
────────────────
国試ベース症例を使う場合も、そのまま見えないように軽く調整する。

【調整してよいもの】
- 年齢を少し変更
- 性別を必要に応じて変更
- 現場背景の表現
- 発見状況や描写の言い回し
- 画像構図

【変えないもの】
- 病態の本質
- 正答につながる重要所見
- 学習価値のコア

12. 今回発生したミスと今後の禁止事項
────────────────
今回、画像作成で以下の誤りが起きた。

【誤り】
- 問題文そのものを画像化してしまった
- 選択肢や正解表示を含んだ問題集風画像になってしまった
- Scene1の現場イラストではなく、説明資料のような画像になった

【今後の禁止事項】
- 問題文をそのまま画像にすること
- 正解付き問題画像を作ること
- テキスト入りの説明画像を作ること
- 「臨床推論シミュレーション」などの見出し入り画像を作ること

【今後の正しい対応】
- 現場の一場面だけをイラストにする
- 傷病者中心
- 背景は状況が伝わる範囲で入れる
- 問題文・解答要素は一切入れない

15. 一言メモ
────────────────
今後は、JSONを先に書くより、先にイラストを作ってからJSONを合わせる方針がよい。
このルールで進めることで、scene1の現場感、media整合、国試感の薄まりが全部やりやすくなる。

さらに今回の修正で、
「画像はScene1の現場イラスト。問題文や選択肢は絶対に入れない」
というルールを最重要事項として固定した。
